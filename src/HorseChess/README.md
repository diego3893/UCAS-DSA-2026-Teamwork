# 马踏棋盘

## 1 题面

**【问题描述】**
设计一个国际象棋的马踏遍棋盘的演示程序。

**【基本要求】**
将马随机放在国际象棋的 8×8 棋盘 `Board[8][8]` 的某个方格中，马按走棋规则进行移动。要求每个方格只进入一次，走遍棋盘上全部 64 个方格。编制**非递归程序**，求出马的行走路线，并按求出的行走路线，将数字 1, 2, …, 64 依次填入一个 8×8 的方阵，输出之。

**【测试数据】**
由读者指定。可自行指定一个马的初始位置 $(i, j), 0 \le i, j \le 7$。

**【实现提示】**

一般来说，当马位于位置 $(i, j)$ 时，可以走到下列 8 个位置之一：
$(i - 2, j + 1), (i - 1, j + 2), (i + 1, j + 2), (i + 2, j + 1), (i + 2, j - 1), (i + 1, j - 2), (i - 1, j - 2), (i - 2, j - 1)$

但是，如果 $(i, j)$ 靠近棋盘的边缘，上述有些位置可能超出棋盘范围，成为不允许的位置。8 个可能位置可以用两个一维数组 `HTry1[0..7]` 和 `HTry2[0..7]` 来表示：

| 索引 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **HTry1** | -2 | -1 | 1 | 2 | 2 | 1 | -1 | -2 |
| **HTry2** | 1 | 2 | 2 | 1 | -1 | -2 | -2 | -1 |

位于 $(i, j)$ 的马可以走到的新位置是在棋盘范围内的 $(i + \text{HTry1}[h], j + \text{HTry2}[h])$，其中 $h=0, 1, \dots, 7$。
每次在多个可走位置中选择其中一个进行试探，其余未曾试探过的可走位置必须用适当结构妥善管理，以备试探失败时的“回溯”（悔棋）使用。

**【选作内容】**
- 求出从某一起点出发的多条以至全部行走路线。
- 探讨每次选择位置的“最佳策略”，以减少回溯的次数。
- 演示寻找行走路线的回溯过程。

## 2 实现思路

### 2.1 总体架构

使用**非递归的深度优先搜索 (DFS)** 框架，手动维护一个栈来模拟递归调用，避免系统栈溢出并精确控制回溯行为。

```c
typedef struct {
    int x, y;   // 当前坐标
    int dir;    // 下一步方向索引 (在排序后的候选列表中的位置)
} StackNode;

StackNode stack[64];  // 棋盘至多 64 步
int top = -1;
```

### 2.2 候选选择策略 — Warnsdorff 启发式

对于当前格子的所有合法候选，按两级优先级排序：

1. **主键 — 出路的度数（越小越好）**：Warnsdorff 规则指出，应优先访问从该格出发的下一步可选位置最少的格子。直观理解：边缘/角落的格子出路少，应尽早处理，否则后期无法访问。
2. **次键 — 距离棋盘中心的距离（越远越好）**：度数相同时，优先走靠近边缘的格子，进一步降低后期困死的概率。

```c
// 排序规则
if (degrees[j] < degrees[i] ||
    (degrees[j] == degrees[i] && dists[j] > dists[i])) {
    swap(i, j);
}
```

其中距离中心的度量使用放大的曼哈顿距离，避免浮点：

```c
int centerDistance(int x, int y) {
    return abs(2 * x - 7) + abs(2 * y - 7);
}
```

### 2.3 死路预判（剪枝）

如果某个候选的出度为 0 且尚未走完 63 步，则该格是名副其实的"死胡同"——走进去之后无处可去，必然失败。直接将其从候选列表中排除：

```c
if (deg == 0 && step < 63) continue;  // 剪枝
```

### 2.4 回溯机制

每个栈节点记录 `dir`——上一次在该节点尝试的候选在排序后列表中的位置。回溯后重新计算候选列表（因为棋盘状态被恢复，排序结果与之前相同），从 `dir + 1` 开始尝试尚未测试的候选：

```c
for (int i = cur->dir + 1; i < count; i++) {
    // 尝试下一个候选
    cur->dir = i;
    push((StackNode){nx, ny, -1});
    // ...
}
```

若所有候选均已失败，则继续回溯到上一层。

### 2.5 选作功能实现

#### 多条路线（含"全部路线"模式）

主循环的终止条件由"找到一条路径"改为"找到 `maxSolutions` 条路径或栈空"。当 `step == 64` 时输出当前解，然后通过**回退一步 → continue** 的方式继续搜索下一条路线：

```c
if (step == 64) {
    solutionsFound++;
    printBoard();
    if (maxSolutions == 1) return 1;   // 只要一条, 立即返回

    // 检查用户是否按 Q 中止
    if (checkAbort()) { userAbort = 1; break; }

    // 回溯一步, 继续搜索
    board[stack[top].x][stack[top].y] = 0;
    pop(); step--;
    continue;
}
```

**不限模式** (`maxSolutions = 0`)：程序将搜索直到栈空（即穷尽所有分支）或用户按 Q 中止。

**按 Q 中止**：搜索过程中使用 `GetAsyncKeyState('Q')` 检测按键，按下 Q 即可随时优雅退出，输出已找到的全部结果和统计数据。

**进度汇报**：非可视化模式下每找到一条解输出完整棋盘，每 10 条刷一条进度行：
```
已找到: 30 条 | 探索节点: 472 | 回溯: 385 | 耗时: 0.0 秒
```

**注意**：8×8 棋盘的骑士周游总数约 260 亿条，实际上不可能穷举。不限模式主要用于观察算法能在一个合理时间内找到多少条不同的解。

#### 回溯过程可视化

在可视化模式下，每次前进/回溯时打印：
- **操作类型**：`[前进]` 或 `[回溯]`
- **坐标变化**：`(x1,y1) --> (x2,y2)`
- **候选排名**：当前候选在全部候选中的排序位置及其度数
- **ASCII 棋盘**：`K` 表示骑士当前位置，`*` 表示已走格子，`.` 表示未访问格子

支持随时输入 `q` 退出可视化模式（程序仍会继续搜索到结束）。可视化模式下同样支持按 Q 键彻底中止搜索。

## 3 编译运行方法

### 编译

本程序为纯 C (C99/C11)，使用 GCC 编译：

```bash
cd src/HorseChess
gcc horsechess.c -o horsechess.exe -Wall -Wextra
```

### 运行

```bash
./horsechess.exe
```

程序提供 4 种运行模式：

| 模式 | 说明 |
| :--- | :--- |
| `[1]` | 寻找一条路径 — 使用 Warnsdorff 启发式快速求解 |
| `[2]` | 寻找多条路径 — 指定数量，输入 0 表示不限（搜完或按 Q 中止） |
| `[3]` | 回溯过程可视化 — 逐步演示每一步的决策和回溯 |
| `[4]` | 可视化 + 多条路径 — 结合 2 和 3 |

> **提示**：任意模式下均可按 Q 键随时中止搜索，程序会输出已找到的结果和统计数据。

## 4 算法复杂度

| 指标 | 理论值 | 实测 (8×8) |
| :--- | :--- | :--- |
| 时间复杂度（最坏） | $O(8^{64})$ 枚举全空间 | Warnsdorff 启发式下几乎所有起点**零回溯** |
| 空间复杂度 | $O(64)$ 显式栈 | 64 个 `StackNode` |

Warnsdorff 启发式对 8×8 棋盘极为有效——经测试，全部 64 个起始位置均能在**零回溯或极少量回溯**的情况下找到一条完整路径。

## 5 测试结果

对全部 $8 \times 8 = 64$ 个可能的起始位置 $(i, j), 0 \le i, j \le 7$ 逐一测试，**全部成功找到合法骑士周游**：

```
(0,0) (0,1) (0,2) ... (7,6) (7,7)  → 64/64 PASS
```

每条路径均经过自动验证脚本确认：所有数字 1~64 恰好出现一次，且 63 个连续步均为合法的骑士走法。
